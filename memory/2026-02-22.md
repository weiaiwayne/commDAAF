## Major Finding: Academic Framing Bypass Hypothesis DISPROVEN

### The Investigation
Wayne asked why z.ai and Kimi API dashboards showed no usage for recent AgentAcademy runs. Investigation revealed:

- AgentAcademy runs (Feb 20+) used **OpenCode free proxy** (`opencode/kimi-k2.5-free`) 
- NOT the direct Chinese API endpoints (`zai-coding-plan/glm-4.7`, `kimi-for-coding/k2p5`)
- Evidence: `$0.00` in OpenCode stats, logs showed `providerID=opencode`

### Controlled Study Results
Tested Xinjiang/Uyghur content via direct APIs:

| Test | API | Prompt Type | Result |
|------|-----|-------------|--------|
| 1 | z.ai GLM (direct) | Direct sensitive | ❌ BLOCKED (code 1301) |
| 2 | Kimi (direct) | Direct sensitive | ❌ BLOCKED (high risk) |
| 3 | z.ai GLM (direct) | CommDAAF-wrapped | ❌ BLOCKED |
| 4 | Kimi (direct) | CommDAAF-wrapped | ❌ BLOCKED |
| 5 | OpenCode free proxy | Same content | ✅ WORKED |

**Conclusion:** Academic framing does NOT bypass Chinese LLM content filters. The bypass was infrastructure-level (OpenCode proxy), not prompt engineering.

## Methodological Lessons

1. **Verify API routing** - Check logs to confirm which provider is being used
2. **Cost as signal** - $0.00 when expecting paid API = wrong provider
3. **Don't rush to publish** - Exciting findings deserve extra scrutiny
4. **Replicate with controls** - Explicit provider specification matters
